WHAT IS THIS?
This panel manages your Zadarma VoIP account — allowing users with WiFi-only phones (no cellular service) to make and receive real phone calls via the Zadarma app.

How it works:
1. You create a SIP user here and assign them a virtual number
2. They download the Zadarma app (iOS / Android) — free
3. They log in with the SIP credentials you provide
4. They make and receive calls over WiFi — no cellular needed
5. Calls show your virtual number as caller ID

SETUP GUIDE — WiFi-Only Phone
Step-by-step: get a user with no cellular service making real calls over WiFi in under 10 minutes.
1
Create SIP User (above in SIP Users tab)
Create a SIP login for the user. Use their name as the login (no spaces). Set a strong password. Copy the credentials.
2
User Downloads Zadarma App
User installs Zadarma from App Store or Google Play — it is free. Needs only WiFi to download.
App Store: search "Zadarma" • Google Play: search "Zadarma"
3
User Logs In With SIP Credentials
Open Zadarma app. Tap Login. Enter the SIP credentials you created. App connects automatically over WiFi.
SIP Server: sip.zadarma.com • Login: [sip_login] • Password: [sip_password]
4
Assign Virtual Number (optional but recommended)
Buy a US or MX virtual number from Zadarma (~$4-6/mo). Assign it to their SIP user here. Now people can call that number and it rings in the app — and outbound calls show that number as caller ID.
5
Test — Make a Call Over WiFi
User dials any number from the Zadarma app. Call routes over WiFi. No cellular plan or SIM card needed — just internet connection.
6
Set ZADARMA_SECRET in Railway
To enable live API sync in this panel, add your Zadarma API secret to Railway environment variables. Find it at my.zadarma.com under API settings.
Railway → auditdna-backend-1 → Variables → ZADARMA_SECRET = [your_secret]
